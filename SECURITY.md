# Security Policy

## Supported Versions

Security fixes apply to the latest version on the default branch and the most recent published release.

## Reporting a Vulnerability

Use GitHub private vulnerability reporting when enabled. If it is unavailable, contact the repository owner privately through their verified GitHub profile. Do not open a public issue.

Include the affected file or workflow, impact, reproduction details, and any suggested mitigation. Remove credentials, customer identifiers, and proprietary material.

## Response Process

Maintainers will acknowledge a report as soon as practicable, assess severity, coordinate remediation, and disclose the issue after a fix is available. Timelines depend on impact and maintainer availability.

## Repository Security Expectations

- Pin and review automation dependencies.
- Grant workflows the minimum required permissions.
- Never commit secrets or production data.
- Treat externally supplied links, examples, and artifacts as untrusted.
