# Documentation Review Checklist

## Grammar

- [ ] Spelling, grammar, punctuation, and sentence structure are correct.
- [ ] Tone is clear, inclusive, direct, and appropriate for the audience.

## Accuracy

- [ ] Technical claims are current, scoped, and verifiable.
- [ ] Product versions, deployment models, and assumptions are explicit.

## Architecture Review

- [ ] Architecture statements and diagrams are internally consistent.
- [ ] Important decisions, constraints, and trade-offs are explained.

## Performance Notes

- [ ] Relevant performance implications and measurement guidance are included.
- [ ] Unsupported benchmarks or universal performance claims are absent.

## Security Notes

- [ ] Relevant threats, controls, secure defaults, and privacy concerns are addressed.
- [ ] Secrets, customer data, and proprietary details are absent.

## Interview Questions

- [ ] Questions match the page's learning objectives.
- [ ] Questions test reasoning rather than unsupported trivia.

## Cross References

- [ ] Related topics use descriptive relative links.
- [ ] The page is reachable from the appropriate navigation or index.

## References

- [ ] Claims cite authoritative, canonical, and accessible sources.
- [ ] Source dates and versions are included where they affect interpretation.

## Formatting

- [ ] Heading hierarchy, code fences, tables, images, and Mermaid follow the style guide.
- [ ] Formatting, linting, and spelling checks pass.

## Broken Links

- [ ] Internal anchors and links resolve.
- [ ] External links pass validation and avoid redirects or private resources.
